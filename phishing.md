# mail_security

# 기간 
25/12/28~26/12/04

# 1. 프로젝트 개요 (Project Overview)
프로젝트명: mail_security

목표: 모의 해킹 또는 보안 훈련을 위한 메일 전송 및 추적 시스템 구축 (메일 오픈 및 버튼 클릭 감지 기능 포함)

# 2. 기술 스택 (Tech Stack)
기획된 기술 스택 : 현대적, 확장성이 좋다. 

Frontend: React (사용자 인터페이스, 메일 템플릿 작성 및 대시보드)

Backend: FastAPI (Python 기반, 비동기 처리에 강하여 메일 전송 및 트래킹 로그 수집에 적합)

Database: MySQL (전송 로그, 수신자 정보, 템플릿 저장 등 정형 데이터 관리)

Infra: Docker Compose (FE, BE, DB를 격리된 컨테이너로 관리하여 배포 및 실행 환경 통일)

SMTP 라이브러리로, SMTP Sendmail 라이브러리 사용. 추가적으로 메일 발송 시 발신자의 메일을 웹에서 선택할 수 있어야함할
1회 메일을 발송했을때 1개의 프로젝트로 간주하고 메일 보낸거에 대한 각 사용자의 결과를 확인할 수 있고, 해당 프로젝트를 종료하여 종료된 태스크로 정리할 수 있음.


# 3. 주요 기능 분석 및 구현 전략

[] 1. 메일 전송 (Bulk Email Sending)

기능: 다수 사용자에게 메일 발송

구현 전략: Python의 smtplib 또는 FastAPI-Mail 라이브러리 사용

준비 사항: SMTP 서버(Google Gmail SMTP, AWS SES, SendGrid 등) 계정 및 API 키 확보 필요

-----------------------------------------------------------------------------------------
[] 2. 메일 템플릿 시스템 (Template System)

기능: HTML 형식의 메일 템플릿 저장 및 재사용

구현 전략

DB: 템플릿 제목, HTML 본문, 생성일 등을 저장하는 테이블 설계

FE: React에서 WYSIWYG 에디터(Quill, Toast UI Editor 등)를 연동하여 HTML 작성 기능 구현

BE: Jinja2 템플릿 엔진을 사용하여 수신자 이름 등을 동적으로 치환({{ name }})하여 발송

-----------------------------------------------------------------------------------------
[] 3. 메일 오픈 추적 (Open Tracking)

기능: 수신자가 메일을 열었는지 확인

구현 전략: Tracking Pixel 방식 사용

메일 본문에 1x1 투명 이미지 형태로 삽입

이미지가 로드될 때 서버가 요청을 받아 해당 ID의 '읽음 상태'를 DB에 업데이트

-----------------------------------------------------------------------------------------
[] 4. 링크/버튼 클릭 추적 (Click Tracking)

기능: 본문 내 버튼 클릭 시 감지

구현 전략 - Redirect 방식 사용

메일 내 버튼 링크를 최종 목적지가 아닌 따로설정

서버는 클릭 로그를 DB에 저장한 후 사용자를 실제 목적지(target)로 리다이렉트

-----------------------------------------------------------------------------------------
[] 5. (심화) 수신자별 상세 추적 (Advanced Tracking)

기능: 누가, 언제 확인했는지 특정
구현 전략: 발송 시점마다 각 수신자에게 고유 UUID를 생성하여 DB에 매핑
트래킹 픽셀과 링크에 이 UUID를 파라미터로 포함시켜 발송하면, 요청이 들어올 때 누구인지 정확히 식별 가능
